# wonn-doc-producer

**Role:** Document and presentation production in Wonn's format and standards.

---

## Document Standards

### Visual Identity in Documents
- Primary colors: follow wonn-brand-dna color system
- Typography: consistent with brand system
- Logo: always top-left on cover, never distorted
- White space: generous — Wonn documents don't look crowded

### Document Types

| Type | Purpose | Format |
|---|---|---|
| Pitch deck | Investor or partner presentations | Slides, 12-18 pages |
| Product brief | Internal feature definition | Notion or PDF |
| Partner proposal | Commercial partnership outlines | Slides or PDF |
| Onboarding guide | New user or partner documentation | PDF or web |
| Press kit | Media and PR materials | PDF + assets folder |

---

## Presentation Standards

**Slide principles:**
- One idea per slide
- Headline states the point (not the topic)
- Max 40 words of body copy per slide
- Data visualized, not listed

**Cover slide:**
- Wonn logo
- Document title
- Date and version
- Recipient name (for personalized decks)

**Deck structure:**
```
Cover → Situation → Problem → Solution → Product → Proof → Ask/Next step
```

---

## File Naming Convention

```
wonn-[document-type]-[audience]-[date]-v[version]
wonn-pitch-investors-2026-03-v1
wonn-proposal-partner-acme-2026-03-v2
```

Always version. Never send "final_FINAL_v3_updated.pdf."

---

## Handoff Checklist

Before any document leaves:
- [ ] Brand colors correct (not approximations)
- [ ] Logo correct version, correct placement
- [ ] All copy proofread
- [ ] All data sourced and accurate
- [ ] File named correctly
- [ ] Exported in correct format for recipient
